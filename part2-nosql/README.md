# Part 2: NoSQL Database Analysis (MongoDB)

## Overview

Part 2 analyzes the limitations of relational databases for diverse product data and demonstrates how MongoDB can be used as a NoSQL solution.

---

## Task 2.1: NoSQL Justification Report

### Description
This task explains why a NoSQL database like MongoDB is suitable for handling highly diverse and semi-structured product data.

### Key Topics Covered
- Limitations of RDBMS for flexible product attributes
- Benefits of MongoDB such as flexible schema and embedded documents
- Trade-offs of using MongoDB compared to relational databases

### File
- `nosql_analysis.md`

---

## Task 2.2: MongoDB Implementation

### Description
This task implements basic MongoDB operations using a sample product catalog.

### Key Activities
- Loaded product data from JSON into MongoDB
- Queried products based on category and price
- Performed aggregation on embedded reviews
- Updated product documents with new reviews
- Calculated average price by category

### Files
- `mongodb_operations.js`
- `products_catalog.json`

MongoDB was successfully used to handle flexible product schemas, nested reviews, and aggregation queries.
